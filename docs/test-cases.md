# 四色牌验收用例（当前单人练习版）

当前可验收范围是单人练习模式。好友同桌、联机匹配、账号登录不在当前可验收范围内。

## 规则逻辑

| ID | 用例 | 步骤 | 预期 |
|---|---|---|---|
| R-01 | 弃牌区全公开 | A主动弃出“黄马”或抓后过出“黄马” | B/C/D均看到A弃牌区顶部出现“黄马” |
| R-02 | 吃操作前置 | B待响区“红将”，A手有“红士”“红象”，集体询问无人响应 | A进入模式1后，“吃”按钮点亮 |
| R-03 | 面板双状态 | A待响区初始牌，无人响应，A点“抓” | 面板切换为“胡/开/碰/吃/过” |
| R-04 | 抓与过语义 | 模式1点“抓”，模式2点“过” | 抓：抓新牌并重新进入集体询问；过：新牌入A弃牌区并移至下家待响区 |
| R-05 | 单将胡牌校验 | 手牌“红将+红士+红象”，待响区“红炮” | 胡失败（不可拆解） |
| R-06 | 抓后再询问 | `local_upper` 阶段点“抓” | 新抓牌进入 `collective` 询问；无人响应后进入 `local_draw` |
| R-07 | 碰/坎结算区分 | 胡牌响应牌 + 手牌两张同面牌组成三张 | 结算按“碰”展示和计分，不按暗坎 |
| R-08 | 胡牌最高分拆解 | 同一批牌存在低分/高分两种合法拆解 | 结算选择高分拆解 |
| R-09 | 开局手牌遮挡 | 单人练习开局发牌动画期间 | 不应先显示完整手牌再播放发牌动画 |

## 前端专项

| ID | 用例 | 步骤 | 预期 |
|---|---|---|---|
| F-01 | 横屏强制 | 手机竖屏打开页面 | 显示“请横屏”全屏遮罩，游戏不可操作 |
| F-02 | 按钮点亮 | B待响区“黄车”，A手无匹配 | A界面“胡/开/碰”灰显，保留可执行按钮 |
| F-03 | 弃牌区展示 | A连续弃3张牌 | B/C/D均看到A弃牌区完整3张牌面 |
| F-04 | 明示区高亮 | A完成一次吃牌 | 响应牌带金边与星标，高亮动画可见 |
| F-05 | 触屏优化 | 手机点击“吃”按钮 | 按钮尺寸不小于48px，点击无明显误触 |
| F-06 | 模式入口 | 进入大厅查看模式卡片 | “单人练习”可用；“好友同桌”“联机匹配”显示即将开放且不可点击 |
| F-07 | 声明面板 | 开局发牌后进入声明阶段 | 可选择鱼、声明暗坎数量，四家确认或倒计时结束后进入对局 |
| F-08 | 对局桌面布局 | 横屏桌面/手机不同宽高比下进入 `playing` | 不显示顶部栏；左右玩家剩余牌不遮挡头像；左右明示区向桌面内侧扩展或换行，不越出屏幕；所有牌使用统一比例 |

## 性能与稳定性

| ID | 指标 | 目标 |
|---|---|---|
| N-01 | 状态同步延迟 | <= 100ms |
| N-02 | 操作响应时间 | <= 200ms |
| N-03 | 首屏加载 | <= 3s |
| N-04 | 横屏切换提示 | <= 1s |
