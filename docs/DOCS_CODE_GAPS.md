## 需要你确认的规则/实现歧义

### 1. 抓之前那张“上家牌”应该记到谁的弃牌区

当前代码：
- 上家打出的牌已经在上家的 `discardPile` 中。
- 本家在 `local_upper` 阶段执行“抓”时，代码不会把这张旧牌再加入本家的弃牌区。
- 若本家随后对抓出的新牌也不吃，`local_draw + pass` 会把这张新牌加入本家的弃牌区，并视作打给下家。

旧文档曾写：
- `P点“抓” -> X入P弃牌区`

需要决定：
- 规则上，抓之前那张上家牌是否应该进入本家弃牌区？
- 如果应该进入，需要改代码；如果不应该，旧文档已经按当前代码口径继续收敛。

### 2. 摸到/抓到将或金条后强制收牌的展示区命名

当前代码：
- 将/金条不能主动打出。
- `local_draw` 阶段遇到将或金条会强制收下并进入弃牌阶段。
- 公开状态里同时存在 `generalArea`、`wildcardPool`、`exposedArea` 等历史字段，其中 `wildcardPool` 目前只是兼容展示，不再参与万能替代。

文档仍有历史叫法：
- 有的地方写“移入 MeldArea（明示区）”。
- 有的地方写 `WildcardPool`。

需要决定：
- 强制收下的将/金条在规则语言里应统一叫“将牌区 / 金条区 / 明示区”中的哪一个？
- 决定后可以继续把字段名、UI 文案和文档统一。
